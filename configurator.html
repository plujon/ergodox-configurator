ErgoDox Layout Configurator (Assembly Instructions)

This tool generates the Teensy board firmware .hex file for controlling the ErgoDox (see instructions).
  • Left click a key in the configurator, then press any keyboard key to assign a key to that position
  • Right click a key in the configurator to assign keys by name
  • Download Current Layout (.hex) will compile your layout to a .hex file which you can load onto the ErgoDox
  • Share Layout will generate a link to your layout for sharing or collaboration
  • Defaults allow you to load preset key configurations
See Additional Information for more details. Special thanks to Ben Blazak's work which inspired this tool.
We're very excited to see what you'll come up with. Remember to share your layouts in the discussion!

    So all you're trying to do is get a layout that you created on the website onto your keyboard? The link SubGothius posted should have all the information you need; but I'll rephrase it a bit below, just in case.

    If you're sure you're logged in to the site (as SubGothius mentioned), and you've reloaded the page (as eviltobz mentioned), you should be able to click the "Download Current Layout (.hex)" button on the Configurator page:

    [pic]

    It should say "Downloading..." for a few seconds, and then a download should pop up in your browser, which you should "Show in Finder" (or "Explorer" on Windows).

    Open up the Teensy Loader application (which you mentioned you've downloaded), and click the "Auto" button (so it turns bright green):

    https://d3jqoivu6qpygv.cloudfront.net/img_bucket/ergodox/image06.png

    Drag and drop the '.hex' file (which should have a name something like "user_RXF8P4_20130610131457_ergodox_firmware.hex") that you just downloaded on the the Teensy loader window: The bottom of the window will now display the name of the file you just dropped onto it.

    Press the little black button on the top right of the Teensy itself:

    https://d3jqoivu6qpygv.cloudfront.net/img_bucket/ergodox/image03.jpg

    The Teensy Loader application should display the progress of loading the file onto the Teensy.

    The three LEDs on the keyboard should light up in sequence from top to bottom while the keyboard is restarting itself, and your keyboard will now be running with the layout you created.

    • benblazak @ JohnGB
    • Apr. 15, 2013
    •
    Unicode characters are not supported in firmware - and can't be, as far as I've been able to see. The people that designed the spec for USB keyboards decided to have them send scan codes (just like keyboards have since the beginning of time, I think) instead of UTF-8 characters. I'm kind of mad at them for that too, lol.

    Also, to those wanting international layouts: the USB spec doesn't define those characters explicitly - it defines a more or less US layout + some "international" keys and other things, and expects the rest to be handled in software (for the convenience of hardware manufacturers). So, good news is that your layout is supported by the firmware. Bad news is that I can't tell you how to do it without resorting to Unixish programs and modifying C ;) - hopefully Jasper's able to get the configurator to detect the keys (though, testing for the detection of keys on keyboards not readily available in the US probably isn't the easiest thing to do - something to bear in mind), or someone else has easier ideas for determining the keycodes and looking up their names in the configurator.

    @Hamid:
    The only way I can think of to do that (without writing a bit of C) is to make the "shift" keys into layer keys - and then you could assign whatever values you wanted to the "shifted" state of each key. The downside of that is that then you would *have* to assign the values you wanted to the "shifted" state of each key - even for the keys you want to be shifted normally. And since there aren't any macros at the moment, this setup would also cost you the ability to press a literal "shift" on layer 0 (if that matters...).

    This limitation is due to the fact that the USB Boot Keyboard specification uses 1 keycode for the "2 @" key, and 1 keycode for the "shift" key, and requires the computer (not the keyboard) to determine what symbol to generate when the "2 @" key is pressed, based on whether the "shift" key is pressed at the same time.

    Actually, if you only want to remap a few shifted keys, and you don't have to move between computers frequently, you might have an easier time remapping those keys in software. Whatever's easiest... :)

Layers

ErgoDox operates on a layer stack principle to determine which keys are sent. The top-most layer of the stack is the active layer, and in the initial (just plugged in) state, only Layer 0 is on the stack. Pushing a layer onto the stack will put that layer on the top of the stack, while popping a layer will remove that layer from the stack (no matter where it is, which makes it a little bit of a misnomer). Toggling a layer will temporarily push that layer onto the stack while the assigned key is pressed. These layer functions can be accessed through the right-click menu.
NOTE #1: Remember to provide the ability to pop a layer off the stack once you push it on top. Otherwise, you could get stuck on a layer and you will need to unplug/plug your keyboard to reset.
NOTE #2: A special Clear Layer Stack key is available that will empty the stack (setting Layer 0 as the active layer).

Transparent Keys

When pressed, these keys will move down the stack looking for a non-transparent key to send, hence the name. They cannot be assigned to Layer 0.

Teensy Bootloader

This is a special key that triggers the Teensy bootloader. This is equivalent to pushing the small button the Teensy board.

Stuck Keys/Unresponsive Keyboard

If a key becomes stuck or your keyboard becomes unresponsive, easiest fix is to unplug/plug your keyboard, and reprogram the Teensy with patched firmware.

Media Keys

Unfortunately, media keys are not working on Windows at this time. There are however, software solutions, and registry solutions for the brave, to overcome this limitation.

Shifted Keys

These are simply keys that normally require shifting to send.

Numpad Keys

These keys behave like normal numpad keys in that they are affected by the numlock state.
